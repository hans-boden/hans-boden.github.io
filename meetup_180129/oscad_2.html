production flow

idea design/ modeling translate the Model into STL code (design/modeling software) translating STL into GCode Slicer-Software (CURA, Slic3r) Translate GCode into electrical signals for Motors (3D-Printer Firmware)

OpenSCAD is essentiall a model description language with a STL-code compiler and a 3D-rendering engine.

Advantages of a script based model:

parameterized can be commented can easily be shared, also in parts can be versioned

powerful libraries available (e.g. tooth-wheel) exact results repeatable iterative development

advantages of OpenSCAD

its open source and its free.

It runs on windows, mac amd linux

it has an active community of developers and users. There are many examples and tutorials online.

it simply works. you can get your first results in minutes.

When the built-in script language is not enough - switch to Python. There are Libraries available, which allow to do the same job in the Python Syntax

What comes in conveniently is the autorefresh feature of Openscad. Whenever the script text changes, OScad automatically refreshes the Display.

A model is created from a few basic objects, which are transformed and combined in a number of powerful operations.

To achieve a certain result, there are usually different ways to combine the elementary objects. There may be differences in the performance

Internal library: Computational Geometry Algorithms Library (CGAL)

OpenCSG is a library that does image-based CSG rendering using OpenGL. CSG is short for Constructive Solid Geometry and denotes an approach to model complex 3D-shapes using simpler ones.

problems:

to a certain degree, you should be comfortable with numbers and math.

complex models take time for the calculation (may take (many) minutes)

When working with modules and variables, it is sometimes difficult to find meaning ful names. I real life we dont use names for the pieces of the machines around us.

The language may appear too limited for larger / more complex problems. Python might save your day.
