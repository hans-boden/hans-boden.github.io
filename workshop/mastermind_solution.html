Python Exercise - The Mastermind Game¶

This text was started as a response to the contribution of Luis. He sent me his solution for the mastermind game and it worked well. My comments on his code turned out to become a complete lesson of how to use Python and some basic rules to produce nice, readable code.

Please take your time and work through this exercise. Thank you all.

Thanks to Luis¶

Dear Luis,

Thank you very much for your code. The Mastermind program works fine, and as it is the only solution I received for the mastermind task, it is definitely the best solution. In the real world with the permanent pressure on time and cost, I would say: ok, good enough is perfect.

But we are here for learning, and I would like to give some proposals, how the code may be improved. Its actually only very few points I will mention here, but they will make a big difference.

I hope you allow me to present your code here, because it is a good starting point to repeat and extend our knowledge about Python. I spent quite some time on commenting and editing it and it would be a pity not to share the work.

The original code:

# python3
"""

import random

class D():

def handle_game(answer,guess):
    answer_s=str(answer)

    aux=[0, 0, 0, 0]

            answer_s=answer_s.replace(answer_s[i], "9", 1)

            pass

            if aux[i]==1:

Some basic Rules¶

Just a few reminders of what I have ‘hopefully’ stated before.

  • Functions should have as little side-effects as possible. A function should return values, but leave the environment unchanged.
  • We use global variables for constant values. The modification of global values is justified only if there is a good argument for it, like simplifying the code by not passing around the same values again and again.
  • Some while loops may have a condition, which is obvious in its context. Often it is easier to understand and to code to use while True ... break
  • It’s important to find a good representation (model) for the problem to solve. Sometimes it is a matter of taste, but often a better representation makes the code shorter and easier to read.

Observing the first rule, already satisfies the second rule.

Start the main() function with:

secret = answer_generator()

The answer_generator() just returns a value instead of storing it globally.

Then use a while loop:

while True:
    user_seq = user_guess()  # return a value, good

And then again:

    output = handle_game(secret, user_seq)  # return value instead of global variable
    result_msg = '{}  ->  {}'   # no parenthesis needed
    print(result_msg.format(user_seq, output))
...

In the handle_game() function you coded:

...
if output == '++++':
    pass
else:
    ....

Its completely legal to return a result, as soon as we have it:

...
if output == '++++':
    return output      # no more checks required, so just return
...

This saves us a level of indentation for the following code.

The user_guess() function illustrates the ‘while True’ advantage even better:

def user_guess():
    while True:
        response = input()   # no argument needed, default is ''
        if ...:
            return response
        print("wrong input...")

We did not touch the regular expressions in the workshop. I personally try to avoid them, but I admit, they are sometimes quite useful. You used this expression:

if  len(str(response)) == 4
and response.isdigit()
and re.match("^[1-6]*$", str(response))

The input() function always returns strings, no need to str() anything. The second condition is not needed, when you have the third condition, which already allows only digits. So lets use all of the available regular expression syntax and write:

if re.match(re.match("^[1-6]{4,4}$", reponse)):
    ...

which matches only exact 4 of the allowed digits. Done!

If you think of using the other tests first, to avoid a performance issue with the regular expression, then you solve a problem which you don’t have. If you really find out, that the regular expression gives you a performance penalty, do the RE-compile thing:

The sequence:

prog = re.compile(pattern)
result = prog.match(string)

is equivalent to:

result = re.match(pattern, string)

You define globally (only once):

four_digits = re.compile("^[1-6]{4,4}$")

and then in your user_guess() function:

if four_digits.match(response):
    ...

Its All about Modeling¶

Until here, everything is easy stuff, and I did not touch much of the code. If you look at your code, specially the answer_generator and the loops in the handle_game() function ... does it look nice? easy to understand? easy to change?

Imagine: we want to use letters instead of digits and we want to use a secret code with 5 symbols ... Not easy to solve. Let’s look at the last rule: find a good representation!

With some experience you will get these points almost instantly: Lets use characters (strings) instead of numerics and let’s use lists instead of strings.

The task at hand does not force us, to use numeric values, so we use characters and strings. Strings are immutable but already are ‘sequences’. When we need to change a sequence, we can do it easier with lists.

Let’s start with the global values:

class G():
    secret_len = 4
    valid_chars = '123456'

Ready for another exercise in string formatting? Here it is:

A Little Exercise in String Formatting¶

The regular expresion from above is: "^[1-6]{4,4}$"

It could be more flexible to adapt to different char sets and different lengths. Insert the valid chars is easy: "^[{}]$".format(valid_chars)

But hey, there are curly braces required as literal characters and we need them also to insert format values. The solution: use double {{ }}, which will appear as single { } in the final string:

"^[{}]{{{},{}}}$".format(valid_chars, secret_len, secret_len)

Same argument used twice? Ugly! Use indexed substitution:

"^[{0}]{{{1},{1}}}$".format(valid_chars, secret_len)

or keyed substitution, whatever you like more:

"^[{chars}]{{{len},{len}}}$".format(dict(chars=valid_chars, len=secret_len))

So we add to the G class:

valid_input = "^[{0}]{{{1},{1}}}$".format(valid_chars, secret_len)

and also:

solved = '+' * secret_len  # adapt to different length

Short, then even shorter¶

Now simplify the answer_generator():

def answer_generator():  # make a secret string
    seq = []
    for _ in range(G.secret_len):  # '_': we dont care for the value
        seq.append(random.choice(G.valid_chars))  # the valid_chars string is iterable
    return ''.join(seq)

or, even shorter:

seq = ''
...
    seq += random.choice(G.valid_chars)
return seq

This is also possible, but is not really for Python beginners. I mentioned the list comprehension and it is in the OOTS documents. It is a good idea, to study the comprehension concept of Python:

def answer_generator():
    return ''.join([random.choice(G.valid_chars) for _ in range(G.secret_len)])

This would eliminate the entire answer_generator, we can use this expression in the main() routine directly

The Power of Lists¶

Now, in handle_game() we replace strings with lists:

def handle_game(answer, guess):
    answer = list(answer) # answer now is a list of characters
    guess = list(guess)   # same for guess, no need to assign new names

for loops can iterate directly over a string or a list. And where we need both, the character and its position, we use the enumerate() function:

output = ''
for i, ch in enumerate(guess):
    if ch == answer[i]: # same position
        guess[i] = '°'  # any char which is not in the list of valid symbols
        answer[i] = '^' # eliminate also on the answer list
        output += '+'

We have now identified all ‘+’ positions and neutralized the characters from both lists. Most of the time, the guess is incomplete, so we continue with the second part and skip the complete-test:

for ch in guess:
    if ch in answer:
        # remove hit from answer, position is no longer important
        answer.remove(ch)  # remove one ch, wherever it is found
        output += '-'

return output  # simply return the output, no check here

The final Proposal¶

Here we are. Most changes are easy to follow. Decide by yourself, if the code is actually easier to read and to understand:

# python3
"""
    Mastermind without kivy - by Luis
    merciless edited by Hans
"""

import random
import re

class G():
    valid_chars = '123456'
    secret_len = 5
    solved = '+' * secret_len
    regex_str = "^[{0}]{{{1},{1}}}$".format(valid_chars, secret_len)
    valid_input = re.compile(regex_str) # regular expression for user input

def main():
    secret = answer_generator()
    print('Enter your guess of {} of these symbols: ({})'
          .format(G.secret_len, G.valid_chars))

    while True:
        user_seq = user_guess()
        output = handle_game(secret, user_seq)

        result_msg = ('{}  ->  {}')
        print(result_msg.format(user_seq, output))
        if output == G.solved:
            break

    print('You have found the answer! Goodbye!')

def handle_game(answer, guess):
    answer = list(answer)  # no need to str() or to assign a new name
    guess = list(guess)

    output = ''
    for i, ch in enumerate(guess):
        if ch == answer[i]:
            # eliminate hits from both lists, but leave position untouched
            guess[i] = '°'   # any char which is not in valid_chars
            answer[i] = '^'
            output += '+'

    for ch in guess:
        if ch in answer:
            # remove hit from answer, position is no longer important
            answer.remove(ch)
            output += '-'

    return output

def user_guess():
    while True:
        response = input()   # no argument needed, default is ''
        if G.valid_input.match(response):
            return response
        print("wrong input...")

def answer_generator(): # Creates random sequence of n characters
    seq = ''
    for _ in range(G.secret_len):  # '_': we dont care for the value
        seq += random.choice(G.valid_chars)  # valid_chars string is iterable
    return seq

if __name__ == '__main__':
    main()
