PyWS02 - Variables and Values¶

Usage of the Python Console¶

The following exercises demonstrate, how variables and values are handeld in Python. You are invited, to use your Python console and type in the following expression. Before you press enter, you should be aware of the expected result. If you can predict the result for all the examples, then you have probably understood, what goes on. The examples go from simple to difficult. The more difficult examples may make you dizzy and confused. That is normal and nobody’s fault. But you should came back from time to time and try again.

Objects in Python¶

Anything in Python is an object

All objects have a type and an id

Depending on the type, an object can be used in different situations and support different operations

Try this:

type(2) type(‘x’) type(‘xx’) type(3.1417) type(sum) type(int)

An operation like ‘+’ may be defined for some types, but not for others:

2 + 7
2 + float('3.432')
2 + 'abc'
'a' + 'b' + 'c'

There will be much more about objects and types later

Variables in Python¶

A variable is a name bound to one object

There are mutable and unmutable objects

Names can be reassigned

These short rules describe fully the behaviour of variables in Python. Let’s do some exercises:

# variable asignment for immutable variables
a = 7
b = 20
mysum = a + b
mysum
s1 = 'hello'
s2 = 'world'
s3 = s1 + ' ' + s2
s3
s3 *= 3
s3
a = 9
mysum = a + b
mysum

c = a
c
a = 15
a
c
a = a + 1
a
a = a * 3
a
a += 1   # a shorter form
a
a *= 2
a

a = b
a
del b  # deletes only the name
b
a

We already know one value, which can be used as a sequence: string. A string is a sequence of characters:

x = 'hello world'
type(x)
x + x

Sequences support slice operations. A numerical index is a special type of a slice operation:

x = 'hello world'
x[0]
x[1]
x[-1]
x[-2]
x[20]   # invalid index

Now some slices:

x[3:8]
x[0:2]
x[1:-1]
x[:4]
x[-5:]
x[:]

There is a third parameter in the slice operator: a step

x[2:9] x[2:9:2] # every second element x[::-1] # reverse the sequence x # was not modified: a string is immutable

Another sequence type: tuple. A tuple is an immutable sequence of arbitrary values:

(1,2,3)
('a', 'xxx', 7.3)
'a', 'xxx', 7.3    # depending of the context, the parenthesis may be omitted
tup = 2,3,4        # this is 'packing' of a tuple
a,b,c = tup        # 'unpack' a tuple

Automatic packing and unpacking of tuples occurs all the time and is a very useful feature of Python

tup = 10, 20 a, b = tup a, b a, b = b, a # packing and unpacking in on step a, b

All values, which we used so far are immutable values. Once created, there is no way to change them. Not let’s move to a secon group: Mutable values.

Mutable values in Python¶

One of the most used data type in Python is the ‘list’. A list is an ordered sequence of values. Orderd does not mean sorted, but the fact, that every element in a list has a defined position. Lists are mutable. What this means will be recognized in the following examples:

l = [1,2,3]
l[0]
l[-1]
t = tuple(l)
t
l[0] = 'hello'
l
t[0] = 'hello'
l.append(4)
l.append(5)
l.extend([8,7,6])
l
l.sort()
l[0] = 99
l.sort()
l
