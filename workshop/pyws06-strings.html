What is a String¶

A String is a sequence of Characters. A string can have a length of 0, 1 or any larger size. A string of length 1 is also called a character, but there is no datatype for character. So every character is essentially a string, too.

As an immutable sequence, the string type supports numerical indexing and slicing (like a tuple).

A string supports the len() function and the sorted() method (which returns a sorted copy), but not the sort() method (because it is immutable).

Some of the documention talk about the characters of a string as of ASCII characters. This is not true for Python3, because all strings are unicode, and the elements of a string are unicode characters.

Strings support the in operator, like this:

s1 = 'Welcome'
'come' in s1  ==> True
'wel' in s1  ==> False  # characters are case sensitive

When (and only when) we need the position of a substring, we use the find and rfind methods:

s2 = 'python workshop'
s2.find('o')
p1 = s2.find('o')
p2 = s2.find('o', p1+1)
p3 = s2.find('o', p2+1)
p4 = s2.find('o', p3+1)
p1, p2, p3, p4  ==> (4, 8, 13, -1)  # returns the position or -1, if not found

To do the reverse sequence, use rfind:

p4 = s2.rfind('o')
p3 = s2.rfind('o', 0, p4)
p2 = s2.rfind('o', 0, p3)
p1 = s2.rfind('o', 0, p2)
p1, p2, p3, p4  ==>  (-1, 4, 8, 13)

the s.index() method works the same, but returns an IndexError

This is powerful:

s3 = s2.replace('o', 'ilo')
s4 = s2.replace('o', 'ilo', 2)

s2.replace('o', 'ilo')
==>  'pythilon wilorkshilop'
s2.replace('o', 'ilo', 2)
==>  'pythilon wilorkshop'

Strings and characters can be ordered (sorted) and support the < and > operators, according to their ordinal value, which is now a unicode number. The the ord() function is reverse to the chr() function:

ord('a')  ==>  97
ord('z')  ==> 122
ord('á')  ==> 225
chr(227)  ==> 'ã'
chr(199)  ==> 'Ç'

There are many(!) methods defined for the string type. See the documentation in the Standard library Built-in types section.
